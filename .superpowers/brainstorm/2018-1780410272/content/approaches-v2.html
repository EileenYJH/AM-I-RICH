Native App vs PWA

Adding PWA to the mix — it changes things quite a bit.

A

PWA (Progressive Web App) ⭐ Recommended for you

A website that installs on your iPhone home screen and looks like an app. Built with React + Claude Vision API. No App Store, no $99/yr, no Mac needed.

Pros

  • No Apple Developer account ($99/yr saved)
  • Deploy free to Vercel or Netlify
  • Build entirely on Windows
  • Instant updates — just redeploy
  • Claude Vision reads any bank app screenshot
  • Simplest stack — just React + a web server

Cons

  • ⚠️ No iOS Share Extension — can't intercept the screenshot button directly
  • Flow becomes: take screenshot → open app → pick from Photos
  • Safari has stricter PWA limits than Android
  • Looks slightly less "native" than a real app
B

React Native + Expo (Real iOS App)

A real iPhone app with full iOS integration. The share extension lets users share screenshots directly from the screenshot preview pop-up.

Pros

  • Share Extension works — screenshot → tap → done
  • Feels fully native on iPhone
  • Can publish to App Store eventually

Cons

  • Requires Apple Developer account ($99/yr)
  • Expo EAS Build setup is more complex
  • Slower to iterate during development

The key trade-off

PWA saves you $99/yr and a lot of complexity, but the screenshot sharing is one extra step — you open the app and pick the screenshot from Photos instead of sharing it straight from the iOS preview pop-up. For personal use, that's usually a fine trade-off. If you want the smoothest possible screenshot flow, go native.
