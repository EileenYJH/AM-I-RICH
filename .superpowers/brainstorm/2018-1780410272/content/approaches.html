How should we build this?

3 approaches — pick the one that feels right. I'll explain the trade-offs below each.

A

React Native + Expo + Claude Vision AI ⭐ Recommended

Build on your Windows PC. Screenshots are sent to Claude's Vision API to intelligently extract balances and transactions. Works for any bank layout — no custom logic per app.

Pros

  • No Mac needed — build on Windows
  • AI reads any bank app screenshot automatically
  • JavaScript — Claude can help you every step
  • Share Extension works out of the box

Cons

  • Small cost per screenshot (Claude API)
  • Needs internet to parse screenshots
  • Needs Apple Developer account ($99/yr) to install on iPhone
B

Native SwiftUI + Apple Vision (On-device OCR)

A fully native iPhone app built in Swift. Uses Apple's built-in OCR engine on-device to read text from screenshots. Free, fast, and works offline — but needs custom rules per bank.

Pros

  • 100% free — no API costs
  • Works offline
  • Best iOS look & feel

Cons

  • Requires a Mac + Xcode to build
  • Harder to code — Swift is less beginner-friendly
  • Need to manually write rules for each bank app layout
C

Flutter + Google ML Kit (On-device OCR)

Cross-platform app using Flutter (runs on Android too). Uses Google ML Kit for free on-device text recognition.

Pros

  • Works on Android too if you ever want it
  • Free, offline OCR
  • Can develop on Windows

Cons

  • Dart language — Claude is less fluent, harder to get help
  • Still needs custom logic per bank layout
  • Less polished iOS feel than native
